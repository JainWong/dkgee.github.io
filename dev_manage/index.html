# 研发经验

记录项目研发管理方面的经验总结。

软件研发领域理解

  1. 技术理论学习 ---> 项目实战运用 ---> 团队建设管理 ---> 理智竞争分析 ---> 行业局势判断 ---> 坚强意志心态。
  2. 好的产品都是深入思考用户业务需求反复迭代出来的。
  3. 目前真正赚钱的还是懂沟通、懂业务的人。现阶段培训或自学出来的技术只是浅层次，达不到独自盈利的地步，只有依赖客户业务生存。技术 要想赚钱，还是得高精端的技术。达不到这个水平，那就探索研究，深入某个行业领域知识。这两种才有可能赚大钱。

# 项目管理

作为一个项目的负责人，对项目的整体把控工具必不可少，但是如何能有效的管理项目的进度和资料呢？就自我而言，项目本身作为信息集合，需要一个合理化的项目信息组织概念。 自己整理了一套项目信息组织目录，作为日后项目管理的参考。

  • XX项目
    1. 项目文档管理：投标材料、立项材料，结项材料，配置文档，开发文档，项目周报，成员绩效。
    2. 项目过程管理（时间周期）：项目投标、用户/产品需求分析（定规格）、系统设计（技术调研、架构设计）、编码实现（数据建模、业务代码[前端、后端、采集、同步流程]）、系统测试（单元/集成）、系统部署（发布版本/上线试用）、交付验收、运营维护（产品运营/系统维护）

合并后，组织目录结构思考列举如下：

参考目录

  • XX项目
    • 归档管理
      • 投标文档
      • 立项文档
      • 配置文档
      • 开发文档
      • 月周报
      • 成员绩效
      • 结项文档
    • 过程管理
      • 项目投标（竞品分析）
      • 用户需求分析（需求调研、需求分析、规格说明）
      • 系统设计方案（技术调研、方案选型、架构设计、业务流程）
      • 系统开发（开发人员、任务分配、模块编码）
      • 系统测试（单元测试、集成测试）
      • 系统部署（发布版本、上线试用、服务器清单）
      • 系统验收（用户手册、交付清单）
      • 系统维护（维护清单、缺陷清单、版本升级）
      • 产品运营（用户业务需求清单、业务数据及文档、3d厂商文档）

通过上述组织目录，要达成以下几点目标：

1. 项目的内外部文档都要有地存，而且能随时找的到；
2. 对于项目的整体质量，要能随时掌握，把控进度；
3. 对于WBS任务分解，要合理规范。
